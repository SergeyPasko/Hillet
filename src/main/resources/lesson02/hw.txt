1. Зробити дві гілки з змінами в однакових рядках файлів.
2. Виконати мердж реквест першої гілки в меін. Push змін
3. Зробити пулл реквест другої гілки використовуючи Github.
4. Впевнитися що є мердж конфлікт (не можемо змерджити автоматом через інтерфейс Github). Скриншот
5. Виконати мердж в другу гілку з бранчі меін. Побачити локально конфлікти.
6. Відкрити тулу для вирішення конфліктів, зробити скриншот.
7. Вирішити конфлікти. Push
8. Змерджити пулл реквест в Github. Скриншот історії комітів з Github
9. Повторити пункти 1-8, в пункті 5* замість merge->rebase
10. Створити нову гілку від мейна. В ній зробити декілька комітів. Скриншот історії комітів
11. Виконати squash для цьої гілки. Скриншот історії комітів


В результаті - 8 скриншотів, по 3 для операцій merge/rebase, 2 squash
